Dec 31, 2017

Hi today we try to understand, how Koa.js works behind the curtains.
We will use the latest version today 2.4.1.
It consists of several composite dependencies, and multiple files.
Let’s get started!

Well, let’s start with the first component ctx or context.
I hope you have experience with ctx, if not, then in simple words it is the object which is passed through the request and contains all the most important things(objects request, response..)

Is next step general module application.js.
Look at the most interesting moments.

Fundamentally in koa js it is middleware and their use.
In documentation is very well write about this.
This is very similar to stack calls.

In app object have is array middlewares, through method app.use you add new middleware and push in array.
Then when base method call (listen), all middlewares processed using helper-package koa-compose.
Koa compose calls one after another middleware and pass in arguments context and the function next, inside the function next , we again function with new middleware from array, and call them one by one.

When finished the stack our middleware will continue run from our point of interruption (will begin to rise in the opposite direction).
